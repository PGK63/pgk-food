# Руководство пользователя — роль REGISTRATOR (Регистратор)

## Роль и доступ

**Что доступно в роли `REGISTRATOR`:**
1. `dashboard` — главный экран регистратора.
2. `users` — список, поиск, фильтрация и управление пользователями.
3. `users_create` — создание нового пользователя.
4. `groups` — создание и сопровождение групп.
5. `settings` — настройки (подсказки, масштаб, уведомления).

## Быстрый старт

### Сценарий LGN-01-REG. Войти и открыть пользователей

**Цель:** начать работу с учетными записями пользователей.

**Перед началом:**
1. Подготовьте логин и пароль.

**Шаги:**
1. Войдите в приложение.
2. Нажмите кнопку `Пользователи`.

**Ожидаемый результат:**
1. Открыт экран списка пользователей с поиском, фильтрами и группировкой по группам.

**Если что-то пошло не так:**
1. Если вход неуспешен, перепроверьте учетные данные.
2. Когда экран пустой сразу после входа, дождитесь первичной загрузки.
3. После ошибки обновления данных нажмите повторную загрузку (перезайдите в экран).

**Автологика системы:**
1. Приложение подтягивает список пользователей и список групп параллельно.

## Основные сценарии

### Сценарий REGU-01. Создать пользователя

**Цель:** зарегистрировать нового пользователя и получить его логин/пароль.

**Перед началом:**
1. Откройте экран `Пользователи`.
2. Убедитесь, что знаете нужные роли и группу.

**Шаги:**
1. Нажмите `+` (FAB `Создать пользователя`) или `+` у конкретной группы.
2. В форме заполните `Фамилия`, `Имя`, `Отчество`.
3. Выберите роли в блоке `Роли`.
4. Если выбрана роль `Студент` или `Куратор`, выберите `Группа`.
5. Если выбрана роль `Студент`, выберите `Категория студента`.
6. Нажмите `Создать`.
7. В окне учетных данных скопируйте логин/пароль.

**Ожидаемый результат:**
1. Пользователь создан.
2. Открыт диалог с сгенерированными логином и паролем.

**Если что-то пошло не так:**
1. Если кнопка `Создать` неактивна, заполните обязательные поля.
2. Когда видите `Не удалось загрузить группы`, обновите экран и повторите.
3. После ошибки `Ошибка создания пользователя` проверьте сеть и повторите.

**Автологика системы:**
1. В форме по умолчанию выбрана роль `STUDENT`.
2. Требования к полям динамические: группа и категория становятся обязательными в зависимости от выбранных ролей.

### Сценарий REGU-02. Найти пользователя по ФИО или логину

**Цель:** быстро открыть нужную карточку пользователя.

**Перед началом:**
1. Откройте экран `Пользователи`.

**Шаги:**
1. Нажмите в поле поиска `Начните искать`.
2. Введите часть ФИО или логина.
3. Откройте найденную запись нажатием на строку пользователя.

**Ожидаемый результат:**
1. Отфильтрован список пользователей.
2. Открывается карточка пользователя (bottom sheet).

**Если что-то пошло не так:**
1. Если отображается `Ничего не найдено`, очистите поиск и проверьте написание.
2. Когда поиск не дает результата по группе, снимите активные фильтры.

**Автологика системы:**
1. Поиск выполняется без учета регистра.
2. Фильтрация применяется одновременно к `name`, `surname`, `login`.

### Сценарий REGU-03. Фильтровать список по группе и роли

**Цель:** сузить выборку пользователей для массовой проверки.

**Перед началом:**
1. Откройте экран `Пользователи`.

**Шаги:**
1. Нажмите чип `Группа` или `Роль`.
2. В нижнем листе `Фильтры` выберите группу.
3. В том же листе выберите роль.
4. Нажмите `Применить`.
5. Для сброса фильтра нажмите кнопку сброса рядом с чипами.

**Ожидаемый результат:**
1. В списке остаются только пользователи под выбранные критерии.

**Если что-то пошло не так:**
1. Если выборка пустая, снимите один из фильтров и проверьте снова.
2. Когда нужно полностью вернуться к общему списку, сбросьте оба фильтра.

**Автологика системы:**
1. Фильтр по группе и роли объединяется логикой `И`.

### Сценарий REGU-04. Изменить роли пользователя

**Цель:** обновить роль, группу и категорию в карточке пользователя.

**Перед началом:**
1. Откройте карточку пользователя.

**Шаги:**
1. Нажмите `Изменить роли`.
2. В диалоге включите/выключите нужные роли.
3. Если добавляете роль `Студент`, выберите `Группа`.
4. Если добавляете роль `Студент`, выберите `Категория`.
5. Нажмите `Сохранить`.

**Ожидаемый результат:**
1. Роли пользователя обновлены.
2. При необходимости обновляется текущая сессия, если изменяли себя.

**Если что-то пошло не так:**
1. Если видите `Для добавления роли студента выберите группу`, заполните поле группы.
2. Когда видите `Для добавления роли студента выберите категорию`, выберите категорию.
3. После сообщения `Роли сохранены, но не удалось обновить текущую сессию` выполните повторный вход.

**Автологика системы:**
1. Жесткая проверка `группа+категория` срабатывает при **добавлении** роли `STUDENT`.
2. После успешного обновления ролей список пользователей перезагружается.

2. Когда копирование не сработало, нажмите копирование повторно в диалоге.

**Автологика системы:**
1. При успешном сбросе пароль действует сразу, старый пароль перестает работать.

**Цель:** добавить новую учебную группу.

**Перед началом:**

**Шаги:**

**Ожидаемый результат:**

### Сценарий REGG-02. Назначить/снять куратора группы

**Цель:** управлять кураторами конкретной группы.

**Перед началом:**
1. Откройте экран `Группы`.
2. Выберите нужную группу.

**Шаги:**
1. Нажмите иконку `Назначить куратора`.
2. В диалоге используйте поиск по ФИО/логину.
3. Выберите пользователя из блока `Доступные к назначению`.
4. Для снятия куратора нажмите иконку `PersonOff` у назначенного куратора.
5. Для полного снятия нажмите `Снять всех кураторов`.

**Ожидаемый результат:**
1. Куратор назначен или снят.
2. Карточка группы обновляет список кураторов.

**Если что-то пошло не так:**
1. Если список пуст, проверьте, есть ли пользователи с ролью `CURATOR`.
2. Когда действие завершилось ошибкой, повторите после обновления данных.

**Автологика системы:**
1. В диалоге исключаются уже назначенные кураторы из блока доступных.
2. После изменения данных выполняется обновление групп и состава.

### Сценарий REGG-03. Добавить/убрать студента из группы

**Цель:** поддерживать актуальный состав группы.

**Перед началом:**
1. На экране `Группы` разверните карточку группы нажатием по ней.

**Шаги:**
1. Нажмите `Добавить студента`.
2. В диалоге при необходимости выберите фильтр по исходной группе.
3. Найдите студента через поиск.
4. Выберите студента в списке.
5. Для удаления студента из группы нажмите иконку удаления напротив его строки в развернутом составе.

**Ожидаемый результат:**
1. Студент добавлен в выбранную группу или удален из нее.

**Если что-то пошло не так:**
1. Если видите `Ошибка добавления студента в группу`, проверьте сеть и повторите.
2. Когда видите `Ошибка удаления студента из группы`, повторите позже.
3. После изменения не видно обновления — сверните/разверните группу или обновите экран.

**Автологика системы:**
1. Для добавления в группу показываются только пользователи с ролью `STUDENT`.
2. Студенты текущей группы исключаются из списка доступных для повторного добавления.

### Сценарий REGG-04. Перевести/переименовать группу с безопасным откатом

**Цель:** перенести группу на новое название с переносом состава.

**Перед началом:**
1. Откройте нужную группу.
2. Убедитесь, что выбрано корректное целевое название.

**Шаги:**
1. Нажмите иконку `Перевести/переименовать`.
2. Введите новое название.
3. Нажмите `Перенести`.
4. Дождитесь завершения операции.

**Ожидаемый результат:**
1. Создана новая группа с новым названием.
2. Студенты и кураторы перенесены.
3. Старая группа удалена.

**Если что-то пошло не так:**
1. Если целевое название уже занято, выберите другое имя.
2. Когда перенос завершился ошибкой, проверьте сообщение: система пытается выполнить откат автоматически.
3. После частичного отката выполните ручную сверку состава групп и повторите перенос при необходимости.

**Автологика системы:**
1. Процедура: создать новую группу -> перенести студентов -> переназначить кураторов -> удалить старую группу.
2. При сбое на любом шаге запускается rollback (возврат студентов/кураторов и удаление временной группы).

### Сценарий REGG-05. Удалить группу

**Шаги:**

**Ожидаемый результат:**

**Если что-то пошло не так:**
1. Если есть сомнения по дубликатам названий, ориентируйтесь по ID и отмените удаление до сверки.
2. Когда сервер вернул ошибку удаления, проверьте связанные сущности и повторите позже.

**Автологика системы:**
1. После удаления выполняется обновление списков групп и связей пользователей.

## Ошибки и нестандартные состояния

1. `Не удалось обновить данные пользователей` — повторите загрузку списка.
2. `Ошибка обновления ролей` — проверьте обязательные поля группы/категории.
3. `Роли сохранены, но не удалось обновить текущую сессию` — выполните повторный вход.
4. `Ошибка сброса пароля` — повторите операцию позже.
5. `Ошибка удаления пользователя` — проверьте сеть и повторите.
6. `Группа с таким названием уже существует` — измените название.
7. `Нет доступа к созданию группы...` — выполните повторный вход и проверьте роль.
8. `Ошибка снятия/назначения куратора` — повторите после обновления данных.
9. `Ошибка добавления/удаления студента из группы` — повторите операцию.
10. `Откат выполнен частично...` при переносе группы — выполните ручную сверку состава.

## Масштаб интерфейса

### Сценарий UI-SCALE. Изменить масштаб

**Цель:** увеличить или уменьшить размер элементов интерфейса.

**Перед началом:**

## Уведомления

### Сценарий PUSH. Включить уведомления в приложении

**Цель:** получать push-уведомления по событиям системы.

**Перед началом:**
1. Откройте `Настройки`.

**Шаги:**
1. Найдите блок `Уведомления`.
2. Переключите тумблер `Включить уведомления` в `ON`.
3. На Android 13+ подтвердите системное разрешение, если появится диалог.
4. Дождитесь завершения обновления (индикатор загрузки под тумблером).

**Ожидаемый результат:**

2. На Android системное разрешение также включено.

**Если что-то пошло не так:**
1. Если видите `Включите системные уведомления в настройках устройства`, откройте настройки ОС и включите уведомления для приложения.
2. Когда видите `Системные уведомления выключены...`, повторите выдачу системного разрешения.
3. После ошибки `Не удалось обновить уведомления` повторите попытку при стабильной сети.

**Автологика системы:**
1. Приложение может один раз автоматически запросить системное разрешение для вошедшего пользователя.

## Подсказки

### Сценарий HINT. Управлять подсказками

**Цель:** включить/выключить обучающие подсказки.

**Перед началом:**
1. Откройте `Настройки`.

**Шаги:**
1. Найдите блок `Подсказки`.
2. Переключите тумблер `Показывать подсказки` в нужное состояние.
3. Откройте нужный экран и при необходимости закройте карточку подсказки через кнопку скрытия.
4. Для полного возврата подсказок включите тумблер снова.

**Ожидаемый результат:**
1. Подсказки отображаются или скрыты в соответствии с настройкой.

**Если что-то пошло не так:**
1. Если подсказка не возвращается на одном экране, включите глобальный тумблер повторно.
2. Когда подсказки мешают, отключите их глобально в настройках.

**Автологика системы:**
1. Без явной ручной настройки подсказки показываются ограниченное время после первого входа (окно 72 часа).
2. Скрытие подсказки на конкретном экране сохраняется отдельно.
3. Повторное глобальное включение очищает скрытия по экранам.

## Автологика системы (почему так)

1. Списки пользователей группируются по `groupId`; для каждой группы доступно быстрые действия (`+`, настройки, удаление).
2. После любой мутации (`create/update/delete`) экран перезагружает данные для консистентности.
3. Для операций с группами используется дополнительная защита от дублей названий и конфликтов прав.
